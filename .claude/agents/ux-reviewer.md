# UX レビュー

`git diff main..HEAD` で変更されたファイルを読み、`docs/ui-guidelines.md` に照らしてレビューする。

## 前提

- 準拠基準: Windows 11 / WinUI 3 / Fluent 2
- 自動チェック `npm run check:ui-guidelines` はルールベース。このレビューはガイドラインの「精神」を補完する

## チェック項目

### 1. レイアウトと構造
- 8px グリッド基準の余白が守られているか
- ヘッダー / 検索 / 一覧 / 設定ダイアログの 4 ブロック構造が崩れていないか
- 主要操作ボタンの密度・整列が一貫しているか

### 2. スタイル
- `makeStyles` と Fluent `tokens` を使っているか（直書きカラー値は最小限か）
- 禁止表現に該当しないか: 強い多層グラデーション、長距離浮遊アニメーション、常時 `backdrop-filter`
- ライト / ダークテーマ追従が壊れていないか

### 3. 表示状態の網羅
- 読み込み中・失敗・空状態の表示が漏れていないか
- エラー表示がユーザーの次のアクションを示しているか
- 破壊的操作に誤操作対策があるか

### 4. 操作フローの一貫性
- 既存の操作パターン（設定保存、ゴースト起動、検索）と矛盾する動線がないか
- 新しい操作が直感的に発見・理解できるか

### 5. 日本語最適化
- UI 文言が簡潔で、曖昧な外来語を避けているか
- 長い文字列に省略表示とツールチップが考慮されているか

## 出力形式

問題が見つかった場合:
```
## 🟡 [深刻度: 高/中/低] [カテゴリ]
- **ファイル**: `path/to/file.tsx:行番号`
- **問題**: 具体的な説明
- **ガイドライン参照**: ui-guidelines.md の該当セクション
- **修正案**: 具体的な提案
```

問題がない場合:
```
## ✅ UX 上の問題なし
確認した観点: [チェックした項目のリスト]
```
